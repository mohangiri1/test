PumpProbeFitting.Fitting module#

class PumpProbeFitting.Fitting.Fitting(num_bootstraps, selected_model_type, Bounds_class, loss='mae', MinMaxScaling=False, track_progress=True, save_file_path='./', plot_within_bounds=False)[source]#

Bases: object

Class for performing data fitting using bootstrap optimization and differential evolution.

Attributes

num_bootstraps

(int) Number of bootstrap iterations for optimization to estimate parameter uncertainties.

selected_model_type

(str) The type of model used for fitting, selected from predefined model options.

Bound_class

(instance, class) Helps in getting the bounds i.e max and min value of the parameters between which optimization is to be performed.

loss

(str) Loss function to use (‘mse’, ‘mae’, ‘chi_square’, ‘tukey’, ‘least_squares’, or ‘ridge’).

MinMaxScaling

(bool, optional) If True, applies Min-Max scaling to the Thz data before fitting. Default is False.

track_progress

(bool, optional) If True, tracks and displays optimization progress. Default is False.

save_file_path

(str, optional) Path to save intermediate results or progress files. Default is ‘./’.

plot_within_bounds

(bool, optional) Default is False, if true, plots the track progress within the bounds.

Notes

  • The function supports multiple pre-defined model types for fitting.

  • Parameter bounds are dynamically defined based on input data (Thz) and are adjusted depending on the model type.

  • The fitting procedure utilizes differential evolution for global optimization within bootstrap iterations to minimize the objective function.

Examples

>>> # Fit a double exponential decay model to simulated pump-probe data:
>>> import numpy as np
>>> t = np.linspace(0, 100, 1000)
>>> Thz = np.exp(-(t-10) / 30) + 0.5 * np.exp(-(t-10) / 10) + np.random.normal(0, 0.1, size=t.shape)
>>> num_bootstraps = 100
>>> selected_model_type = 'double_exp'
>>> Bounds_class = Bounds()
>>> fitting_func = Fitting(num_bootstraps, selected_model_type, Bounds_class, MinMaxScaling= True, track_progress = False, save_file_path = './')
>>> fitted_params, uncertainties, final_model_mean = fitting_func.bootstrap_optimization(t, Thz)

VariableDependentFit(df_experimental, unit, variable_name, variable_value, energy=None, overrides=None)[source]#

Perform variable dependent fitting (eg power dependent or temperature dependent study) for a given variable (e.g., power, temperature, energy).

Parameters:
df_experimentalpandas.DataFrame

The input experimental data with specific column requirements: - Must contain a time_delay [ps] column representing the independent variable (time delay). - Must contain columns named with the format del_T (<value><unit>), where <value> corresponds to the variable (e.g., power or temperature) and <unit> is the unit (e.g., mW, K). - Optionally include an Energy [eV] column for energy-specific data.

unitstr

The unit of the variable. Options include: - ‘mW’ for power-dependent fitting - ‘K’ for temperature-dependent fitting - ‘eV’ for energy - ‘nm’ for wavelength

variable_namestr

Name of the variable for fitting (e.g., “power”, “Temperature”, “Energy”, “Wavelength”).

variable_valuelist or numpy array

A list or array of variable values for which the fitting is to be performed. These values must match the numeric values in the del_T (<value><unit>) column names of the input df_experimental.

energyfloat or int, optional

An optional energy value to include in the resulting parameter DataFrame.

overrideslist of dictionary

An optional dictionary to modify the parameter space for a given model.

Returns:
df_with_resultpandas.DataFrame

A DataFrame containing: - Original experimental data (time_delay [ps] and del_T (<value><unit>) columns). - Fitted values for each variable as new columns named del_T (<value><unit>)_fit. - Energy [eV] (if available) and the selected model type as additional columns.

df_parampandas.DataFrame

A DataFrame containing extracted parameters and their uncertainties for each variable value. Includes: - The variable name and its values. - Fitted parameters and their uncertainties. - Energy (if specified) and the selected model type.

Notes

  • Iterates through the variable_value list and performs fitting for each value.

  • The fitting process uses bootstrap_optimization to extract fitted parameters, uncertainties,

and fitted values. - The SaveResults class is used to generate parameter DataFrames.

class PumpProbeFitting.data_model.Models[source]#

Bases: object

Collection of custom theoretical model functions for fitting data.

This class provides various theoretical model functions for fitting experimental data. These functions cover scenarios such as single exponential decay, double exponential decay, triple exponential decay, and tetra exponential decay, with options for finite rise time in some cases.

A class to evaluate and display goodness-of-fit metrics for a fitted model.

This class calculates the following goodness-of-fit metrics: - Chi-Square (χ²) - Reduced Chi-Square - Akaike Information Criterion (AIC) - Bayesian Information Criterion (BIC) - Coefficient of Determination (R²)

Attributes

actual_data

(numpy.ndarray) Observed data points used for fitting.

fitted_data

(numpy.ndarray) Predicted data points from the fitted model.

fitted_params

(numpy.ndarray) Parameters of the fitted model.

uncertainties

(numpy.ndarray) Uncertainties associated with each fitted parameter.

Methods

Calculate the Akaike Information Criterion (AIC).

BayesianInformationCriterion()

Calculate the Bayesian Information Criterion (BIC).

chi_square()

Calculate the chi-square value.

display_goodness_of_fit()

Display goodness-of-fit metrics in a well-formatted table.

r_squared()

Calculate the coefficient of determination (R²).

reduced_chi_square()

Calculate the reduced chi-square value.

Notes

  • Chi-Square (χ²) and Reduced Chi-Square are absolute measures of goodness of fit.

  • AIC and BIC allow comparison between models, with lower values indicating better fit after penalizing for model complexity.

  • R² is a relative measure often used for quick assessment of fit quality.

References

Based on lmfit documentation: https://lmfit.github.io/lmfit-py/fitting.html

>>> uncertainties = np.array([0.1, 0.05])
